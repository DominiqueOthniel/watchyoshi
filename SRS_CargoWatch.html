1. Introduction

1.1 Document Purpose

This document presents the Software Requirements Specification (SRS) for the CargoWatch system, a real-time shipment tracking web application. It defines the functional and non-functional requirements, architecture, and use cases.

1.2 System Scope

CargoWatch enables businesses and individuals to create shipments, track their progress in real time on a map, manage PDF receipts, view customer reviews, and provide support via live chat.

2. General Description

2.1 Product Perspective

CargoWatch is a monolithic web application (Node.js/Express) that serves both the frontend (HTML/CSS/JS) and backend (REST API). It uses MongoDB for persistence and Socket.io for real-time chat.

2.2 Main Functions

  • Shipment creation and management
  • Real-time tracking on map (Leaflet) with OSRM routing
  • Administrator authentication
  • Multilingual PDF receipt generation
  • Customer reviews system (CRUD)
  • Live chat customer support
  • Swagger/OpenAPI API documentation

2.3 Users

4. Non-Functional Requirements

4.1 Performance

  • API response < 2 s for standard requests
  • Shipment position update every 10 seconds

4.2 Security

  • Hashed passwords (bcrypt)
  • Secure sessions (express-session)
  • Server-side input validation

4.3 Availability

  • Deployment on Render (cloud)
  • MongoDB Atlas database (high availability)

4.4 Maintainability

  • Modular code (JSON/MongoDB separation)
  • API documentation and comments
